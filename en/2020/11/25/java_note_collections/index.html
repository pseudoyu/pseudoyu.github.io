集合类

概述

为了在程序中保存数目不确定的对象，JDK中提供了一系列特殊的类，这些类可以存储任意类型的对象且长度可变，称为集合，位于java.util包中

按存储结构分类

  • Collection
    • List
      • ArrayList：基于动态数组实现，支持随机访问
        • CopyOnWriteArrayList
          • 读写分离，读在原数组，写则是复制到新数组（加锁）
          • 内存为两倍左右
          • 数据不同步
          • 不适合对内存敏感或是实时性要求较高的数组
      • LinkedList：基于双向链表实现（JDK1.6之前为循环链表，JDK1.7取消了循环），只能顺序访问，但是可以快速地在链表中间插入和删除元素，也可以用作栈、队列和双向队列
      • ArrayList和LinkedList区别
        • 都不同步，不保证线程安全
        • ArrayList底层使用Object数组，LinkedList使用双向链表
          • 双向链表包含两个指针，一个prev指向前一个节点，一个next指向后一个节点
          • 双向循环链表最后一个节点的next指向head，而head的prev指向最后一个节点，构成一个环
        • LinkedList不支持高效随机元素访问，ArrayList支持
        • ArrayList空间浪费在list列表结尾会预留一些空间，而LinkedList每一个元素都需要消耗币ArrayList更多的空间
      • Vector：与ArrayList类似，但是是线程安全的
      • RandomAccess接口：标识实现这个接口的类具有随机访问功能，如ArrayList
    • Set
      • HashSet：基于HashMap实现，支持快速查找，但不支持有序性操作，并且没有插入顺序信息（用Iterator遍历的结果不确定）
        • 实现Set接口，仅存储对象，调用add()方法向Set中添加元素
        • 使用成员对象来计算hashcode值
        • LinkedHashSet：具有HashSet的查找效率，并且内部使用双向链表来维护元素的插入顺序。
      • TreeSet：基于红黑树实现，支持有序性操作，查找效率不如HashSet（O(1)），为O(Log N)
      • 无序性是指存储的数据在底层数组中并非按照数组索引的顺序添加，而是根据数据的哈希值决定的
      • 不可重复性是指添加到元素按照equals判断时返回false
      • Comparable和Comparator

        • 一般需要对一个集合使用自定义排序时，需要重写compareTo或compare方法

        • 如果需要对一个集合使用两种排序，可以重写compareTo，使用自制的Comparator方法或者以两个Comparator来实现

          // 定制排序的用法
Collections.sort(arrayList, new Comparator<Integer>() {

    @Override
    public int compare(Integer o1, Integer o2) {
        return o2.compareTo(o1);
    }
});
System.out.println("定制排序后：");
System.out.println(arrayList);

        • 重写compareTo方法实现按年龄来排序

          // person对象没有实现Comparable接口，所以必须实现，这样才不会出错，才可以使treemap中的数据按顺序排列
// 前面一个例子的String类已经默认实现了Comparable接口，详细可以查看String类的API文档，另外其他
// 像Integer类等都已经实现了Comparable接口，所以不需要另外实现了
public  class Person implements Comparable<Person> {
    /**
    * T重写compareTo方法实现按年龄来排序
    */
    @Override
    public int compareTo(Person o) {
        if (this.age > o.getAge()) {
            return 1;
        }
        if (this.age < o.getAge()) {
            return -1;
        }
        return 0;
    }
}
    • Queue
      • LinkedList：可用来实现双向队列
      • PriorityQueue：基于堆结构实现，可以用它来实现有限队列
  • Map
    • Hashtable：类似于HashMap，但是是线程安全的
      • Properties
    • HashMap：基于哈希表实现
      • 多线程下使用容易造成数据丢失等问题
      • 存储结构
        • 包含一个Entry类型的数组table，存放键值对（hashCode，K，V，next）
        • Entry是一个链表，通过拉链法解决冲突
        • 头插式进行
      • JDK1.8开始，桶的链表长度大于8时会转换为红黑树，但如果当前数据长度小于64则会优先选择数组扩容
      • 查找
        • 计算键值所在的桶
        • 在链表上顺序查找，时间复杂度取决于链表长度
      • 允许插入null的键值对，使用第0个桶存放
      • ConcurrentHashMap
        • 使用了分段锁segment，维护着多个HashEntry，因此可以多线程访问
        • 分段锁继承自重入锁ReentrantLock
        • 每个segment维护了一个count变量来统计键值对个数
        • JDK1.8采用了CAS操作来支持更高的并发度，在CAS操作失败时使用内置锁synchronized
      • LinkedHashMap
      • WeakHashMap
        • 继承自WeakReference，被关联的对象在下一次垃圾回收时会被回收
        • 主要用来实现缓存，通过WeakHashMap引用缓存对象，JVM进行回收
        • Tomcat中的ConcurrentCache使用了缓存功能，分代缓存
          • 常用对象放入eden中，ConcurrentHashMap
          • 不常用对象放入longterm，使用WeakHashMap
          • 调用get时，优先从eden中获取，没找到则在longterm中获取
          • 调用put时，如果大小超过了size，则全部放入longterm，JVM回收掉一部分
    • TreeMap
      • 基于红黑树实现
      • 实现了NavigableMap接口和SortedMap接口
        • Navigable接口让TreeMap有了对集合中的元素搜索的能力
        • SortMap接口让TreeMap有了对集合中的元素根据键排序的能力，默认是按照key的升序排序
    • HashMap和HashTable的区别
      • HashMap线程不安全，HashTable线程安全（经过synchronized修饰）
      • HashMap效率高一点
      • HashMap可以拥有一个null键和多个null值，而HashTable不允许，不然会抛出NullPointerException
      • 初始值
        • HashTable创建默认大小为11，每次扩充为2n+1；HashMap默认大小为12，每次扩充为两倍
        • 创建时如果给了初始值，HashTable直接使用，而HashMap则扩充为2的幂次方大小
      • 底层数据结构
        • HashMap在JDK1.8之后解决哈希冲突有了较大变化，当链表长度大于8时，将链表转为红黑树以减少搜索时间
    • ConcurrentHashMap和HashTable的区别
      • HashTable采取数组+链表，而JDK1.8之后的ConcurrentHashMap采取分段的数组+链表/红黑树
      • HashTable使用synchronized来保证线程安全，效率比较低（同一把锁，存在竞争）
      • ConcurrentHashMap采用segment分段锁，每一把锁只锁容器其中一部分数据，多线程访问容器中不同数据段的数据，就不会存在锁竞争；JDK1.8之后摒弃了Segment分段锁，而是直接使用Node数组+链表+红黑树的形式，并发控制使用了synchronized和CAS（CompareAndSet，先进行比较再修改，对应着一条硬件指令，虽然有多个步骤，但是操作系统保证其原子性）来操作
        • synchronized只锁定当前链表或红黑书的首节点，只要哈希不冲突，就不会产生并发，提升效率

Collection接口

通用方法

  • boolean add(Object o) 向集合中添加一个元素
  • boolean addAll(Collection c) 将指定集合中所有元素添加到该集合中
  • void clear() 删除该集合所有元素
  • boolean remove(Object o) 删除该集合指定元素
  • boolean removeAll(Collection c) 删除该集合所有元素
  • boolean isEmpty() 判断是否为空
  • boolean contains(Object o) 判断该集合是否包含某个元素
  • boolean containsAll(Collection c) 判断集合中是否包含指定集合中的所有元素
  • Iterator iterator(Collection c) 返回在该集合的算俗尚进行迭代的迭代起（用于遍历）
  • int size() 获取该集合元素个数

List接口

继承自Collection接口

  • 允许出现重复的元素
  • 线性存储，可通过索引访问集合中的指定元素
  • 元素有序

增加了根据元素索引进行操作的方法

  • boolean add(int index, Object e) 插入元素至指定位置
  • boolean addAll(int index, Collection c) 插入集合至指定位置
  • Object get(int index) 返回指定索引元素
  • Object remove(int index) 删除指定索引元素
  • Object set(int index, Object e) 将索引处元素替换，并返回替换后的元素
  • int indexOf(Object o) 返回元素在List集合中的位置
  • int lastIndexOf(Object o) 返回元素在List集合中最后出现的位置
  • List subList(int fromIndex, int toIndex) 两个索引之间集合组成的子集合，包含头不包含尾

ArrayList

  • 是List接口的一个实现类，底层的存储结构式数组
  • 大部分方法从接口中继承，其中add()方法和get()方法主要用于元素存取
  • 查询元素时效率高，但增删元素时效率低

LinkedList

  • 内部维护了一个双向循环链表

  • 对于元素的增删具有很高的效率

  • 方法

    • void add(int index, E e) 在列表指定位置插入指定元素
    • void addFirst(Object o) 将指定元素插入列表开头
    • void addLast(Object o) 将指定元素插入列表结尾
    • Object getFirst() 返回列表第一个元素
    • Object getLast() 返回列表最后一个元素
    • Object removeFirst() 移除并返回列表第一个元素
    • Object removeLast() 移除并返回列表最后一个元素
    LinkedList list=new LinkedList();

  list.add(9);
  list.add(10);
  list.add(11);
  list.removeFirst();
  list.removeLast();

Iterator接口

  • 用于遍历集合中的所有元素

  • 迭代器获取ArrayList集合中的元素时，会当作Object看待，如想得到特定元素则需要强制类型转换

  • 方法

    • hasNext()
    • next()
    // 构建一个迭代器
Iterator iterator=list.iterator();

// 通过hasNext()方法来循环遍历
while (iterator.hasNext()){
  // 通过next()来访问下一个元素
  Object o=iterator.next();
  list.add(o);
}

foreach循环

for (容器内元素类型 临时变量:容器变量) {
  执行语句
}

List<String> list=new ArrayList();
  list.add("123");
  list.add("234");
  list.add("abc");
  
  for(String p:list){
    System.out.println(p);
    p="999";
  }
  
  for(Object p:list){
    System.out.println(p);
  }

  • 如果要进行remove操作，可以调用迭代器的remove方法而不是集合类的reomve方法，因为如果列表在任何时间从结构上修改创建迭代器之后，以任何方式除非通过迭代器自身remove/add方法，都会抛出一个ConcurrentModificationException，这就是单线程状态下的fail-fast机制

  • Java8开始，可以使用Collection#removeIf()方法删除满足特定条件的元素

        List<Integer> list = new ArrayList<>();
    for (int i = 1; i <= 10; ++i) {
        list.add(i);
    }
    list.removeIf(filter -> filter % 2 == 0); /* 删除list中的所有偶数 */
    System.out.println(list); /* [1, 3, 5, 7, 9] */

Set接口

继承自Collection接口，Collection接口中的方法一致但更加严格

  • 元素无序
  • 保证存入的程序不出现重复

HashSet

  • 根据对象的哈希值来确定元素在集合中的存储位置
    • 向HashSet添加一个对象时，首先调用那个hashCode方法计算对象的哈希值，确定元素的存储位置
    • 如果哈希值相同，调用对象的equals方法确保该位置没有重复元素
  • 具有良好的存取和查找功能

TreeSet

  • 以二叉树的方式来存储元素
  • 可以实现对集合中元素进行排序
  • TreeSet有序指的是元素会按照大小关系进行排序

Map接口

存储具有对应关系的数据

  • 双列集合
  • Key和Value进行对应
  • Key不能重复

常用方法

  • void put(Object key, Object value) 存入键值对
  • Object get(Object key) 获取值
  • boolean containsKey(Object key) 是否包含key
  • boolean containsValue(Object value) 是否包含值
  • Set keySet 返回键的Set
  • Collection values() 返回值的Collection
  • Set<Map.Entry,<K, V»entrySet() 返回映射关系的Set

HashMap

  • 判断key是否重复和HashSet一样 跟hashCode和equals方法相关

LinkedHashMap

  • 维持元素读取和存储的顺序一致

Properties

  • Hashtable是线程安全的 性能比较低
  • Properties是Hashtable的子类
  • 应用场景：多线程安全的需求 往往对应到系统属性

容器中的设计模式

  • 迭代器模式
    • 继承了Iterable接口，iterator()方法能够产生一个iterator对象
    • JDK1.5 之后可以通过foreach来遍历实现了Iterable接口的对象
  • 适配器模式
    • Arrays.asList可以把数组类型转换为List类型

Arrays.asList()使用指南

  • 可以将一个数组转换为一个List集合

    Integer[] arr = {1, 2, 3};
List list = Arrays.asList(arr);
// 另一种方式
List list = Arrays.asList(1, 2, 3);

  • 将数组转换为集合后，底层还是数组

  • 传递的数组必须是数组对象而不是基本类型（泛型方法），因此要使用包装类型数组

  • 使用集合的修改方法add()，remove()和clear()会抛出异常，因为返回的是java.util.Arrays的一个内部类，并没有实现（重写）集合的修改方法，只重写了get, set, indexOf, contains, forEach, replaceAll, sort

正确地将数组转为ArrayList

  • 简便方法

    List list = new ArrayList<>(Arrays.asList("a", "b", "c"));

  • 使用Java8的Stream

    Integer [] myArray = { 1, 2, 3 };
List myList = Arrays.stream(myArray).collect(Collectors.toList());

//基本类型也可以实现转换（依赖boxed的装箱操作）
int [] myArray2 = { 1, 2, 3 };
List myList = Arrays.stream(myArray2).boxed().collect(Collectors.toList());

  • 使用Java9的List.of()方法

    Integer[] array = {1, 2, 3};
List<Integer> list = List.of(array);
System.out.println(list); /* [1, 2, 3] */
/* 不支持基本数据类型 */

Collection.toArray()方法使用

  • 泛型方法，如果toArray方法没有传递任何参数则返回Object类型数组

    String [] s= new String[]{
    "dog", "lazy", "a", "over", "jumps", "fox", "brown", "quick", "A"
};
List<String> list = Arrays.asList(s);
Collections.reverse(list);
s=list.toArray(new String[0]);//没有指定类型的话会报错

  • 由于JVM优化，new String[0]作为Collection.toArray()方法的参数使用，是作为一个模板作用，指定了返回数组的类型，0为了节省空间

泛型

功能

  • 和集合联合使用的时候

    • 限制往集合添加的数据类型
    • 保证获取元素的时候 使用具体的数据类型 而不是Object 从而避免不必要的强制类型转换
    // 创建集合并指定泛型为String
ArrayList<String> list = new ArrayList<String>();
