Java学习笔记 - 面向对象

作者： pseudoyu / 2020-11-23 / 分类： Programming

java, programming

翻译： EN

Java面向对象

特性

  • 封装性：将对象的属性和行为封装起来，不需要让外界知道具体的实现细节
  • 继承性：无需重新编写原有类的情况下，新类可拥有原有类的功能
  • 多态性：一个类中定义的属性和方法被继承后，可以具有不同的数据类型和表现出不同的行为

类与对象

  • 成员变量：定义在类中的变量
  • 局部变量：定义在方法中的变量
class Person {
  // 定义成员变量
  int age = 10;
  void speak() {
    // 定义局部变量，方法优先访问局部变量
    int age = 60;
    System.out.println(age);
  }
}

类的构造方法

  • 方法名与类名相同
  • 方法名前没有返回值声明
  • 不能使用return返回值
  • 使用this来访问成员变量以解决命名冲突
// 无参数默认
public Person() {
  // 构造函数
}

// 重载构造函数
public Person(age n) {
  // 新的构造函数
}

创建对象

// 类名 对象名 = new 类名();
Person p = new Person();

垃圾回收机制

  • 对象成为垃圾后仍然会占用内存空间
  • Java虚拟机自动启动垃圾回收器将垃圾从内存中释放
  • 可以调用System.gc()方法通知虚拟机立即进行垃圾回收
  • 对象在内存中被释放时，finalize()方法会被自动调用，因此可以在类中定义finalized()方法来观察对象何时被释放

封装方法

  • private修饰成员变量
  • 为外界提供set和get方法

static关键字

  • 静态变量：修饰成员变量，特定的数据在内存中只有一份，可以通过类名来访问
  • 静态方法：可以通过类名.方法名的方式来访问
    • 静态方法只能访问static修饰的成员
    • 静态方法不依赖于任何实例，因此必须有实现，不能是抽象方法
  • 静态代码块：static修饰的代码块
    • 类被加载时执行，一次
    • 通常被用来对类的成员作初始化
  • 静态导包
    • 在使用静态变量和方法时不用再指名ClassName，简化代码，但降低了可读性

成员内部类

一个类中除了可以定义成员变量、成员方法，还可以定义类

  • 普通成员内部类
  • 静态内部类
  • 方法内部类
  • 匿名内部类

特性

  • 成员内部类可以访问外部类的所有成员
  • 外部类访问内部类需要通过外部类去创建内部类对象
// 外部类名.内部类名 变量名 = new 外部类名().new 内部类名();
outer.inner var1 = new outer().new inner();

初始化顺序

  • 父类（静态变量，静态语句块）
  • 子类（静态变量，静态语句块）
  • 父类（实例变量，普通语句块）
  • 父类（构造函数）
  • 子类（实例变量，普通语句块）
  • 子类（构造函数）

访问权限

访问级别

  • private 只能被该类的其他成员访问
  • default 能被包中的其他类访问。默认访问控制级别
  • protected 可以被包中的其他类和子类访问
  • public 不管访问类与被访问类是否在同一个包中都可以访问

类的继承

  • 在一个类的基础上构建新的类
  • 子类会自动拥有父类所有可继承的属性和方法
  • 使用extends关键字
  • 不允许多重继承
  • 多个类可以继承自同一个父类

方法重写

  • 子类会自动继承父类中定义的方法
  • 可以对方法进行修改，即重写
  • 重写方法需要和父类方法具有相同的方法名、参数列表及返回类型
  • 使用super方法访问父类成员
  • 使用super方法访问父类构造方法，第一行
  • 使用this关键字调用子类构造方法，第一行

final关键字

  • final修饰的类不可被继承
  • final修饰的方法不可被重写
  • final修饰的变量不可被改变赋值

抽象类

  • 使用abstract关键词修饰
  • 包含抽象方法的类必须声明为抽象类
  • 抽象类可以不包含任何抽象方法
  • 抽象类不可以被实例化
// 定义一个抽象Animal类
abstract class Animal {
  // 定义一个抽象方法shout()
  abstract int shout();
}

接口

  • 接口是由常量和抽象方法组成的特殊类，是对抽象类的进一步抽象
  • 使用接口主要是为了克服单继承的限制
  • 使用extends来继承别的接口
  • 使用interface关键字来声明
  • 类使用implement关键字来继承接口
  • 接口中的变量默认用public static final修饰
  • 接口中的方法默认用public static修饰
public interface 接口名 extends 接口1, 接口2, ... {
  数据类型 常量名 = 常量值;
  返回值 抽象方法名(参数列表);
}

抽象类与接口对比

  • 都不能实例化，都可以有抽象方法
  • 接口可用于多继承
  • 接口具有默认修饰符
  • 关键字不一样
    • 接口继承接口，类继承类用extends
    • 类继承接口implements

使用场景

  • 接口（推荐）
    • 需要让不相关的类实现同一个方法
    • 需要多重继承
  • 抽象类
    • 需要在几个相关的类中共享代码
    • 需要能控制继承来的成员的访问权限
    • 需要继承非静态和非常量字段

多态

父类引用，指向不同子类的实例，对同一个实例方法名调用，调用的程序不一样

  • 多态的基础是继承和重写
  • 多态仅限于实例方法

对象的类型转换

  • 显式转换
  • 隐式转换
// 把子类对象当作父类类型
Animal an1 = new Cat();
Animal an2 = new Dog();

异常处理

异常结构

  • Throwable
    • Error 一般是Java运行时产生的系统内部错误或者资源耗尽的错误，仅靠修改程序本身不能恢复执行
    • Exception 异常 RuntimeException用于运行时异常，其他用于编译时异常
  • Throwable类常用方法
    • String getMessage()
    • void printStackTrace()
    • void printStackTrace(PrintStream s)

异常处理方法

try {
  // 程序
} catch (异常类型类 e) {
  // 对异常的处理
} finally {
  // 一定执行的语句
}

throws关键字

修饰符 返回值类型 方法名() throws 异常类型1, 异常类型2 {
  // 定义方法
}

自定义异常

// 自定义异常类
public class CustomerException extends RuntimeException {
  // 继承构造函数
  public CustomerException() {
    super();
  }

  // 自定义异常方法
}

// 调用自定义异常类
throws Exception 异常对象

代理

代理关系模型，A想要调用B类的方法，A不直接调用，A会在自己的类中创建一个B对象的代理，再由代理调用B的方法

泛型

  • 用泛型表示类
  • 用泛型表示接口
  • 泛型方法
  • 泛型通配符

反射

Java反射机制是在程序运行的过程中，对于任何一个类，都能够知道它的所有属性和方法；对于任意一个对象，都能够知道调用它的任意属性和方法。

  • 在运行时判断一个对象所属的类
  • 在运行是构造任意一个类的对象
  • 在运行时判断任意一个类所有的成员变量和方法
  • 在运行时调用任意一个对象的方法
