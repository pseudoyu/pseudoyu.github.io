Java基础

Java特性

  1. 简单性
    • 丢弃了C++中运算符重载、多重继承等
    • 不使用指针而是使用引用
    • 自动垃圾回收机制（不用开发者进行内存管理）
  2. 面向对象
    • 提供了类、接口和继承等
    • 支持类之间的单继承
    • 支持接口之间的多继承
    • 全面支持动态绑定
  3. 安全性
    • 不支持指针，一切对内存的访问都必须通过实例变量来实现
  4. 可靠性
  5. 跨平台性
    • 可以运行在各种平台上
  6. 支持多线程
    • 并发执行多个任务，提升执行效率
  7. 支持网络编程（Java语言诞生就是为了简化网络编程而设计的）
  8. 编译与解释并存

与C++的区别

  • 都面向对象，实现了封装、继承和多态，但是Java是类不能多重继承（接口可以），C++可以
  • Java不提供指针访问内存，程序内存更加安全
  • Java提供了自动内存管理垃圾回收机制，程序员不需要手动释放
  • Java中没有结束符'\0’而C语言有

import java 和 javax区别

  • 以前javaAPI必需的是java开头的包，javax只是拓展API
  • 后来javax逐渐拓展为javaAPI的组成部分，但移植麻烦，最终成为标准API的一部分

JDK（Java Development Kit）

Java开发环境

  • 编译器
  • 运行工具
  • 文档生成工具
  • 打包工具
  • JRE（Java Runtime Environment）运行环境

Java程序编译

  1. javac编译源文件
  2. 生成.class字节码文件
  3. java虚拟机解释执行

JVM虚拟机

  • 针对不同的操作系统有特定的实现，使用相同的字节码，给出相应的结果
  • 解决了传统解释型语言执行效率低的问题，又保留了其可移植的特性
  • 引进了JIT编译器，运行时编译，完成第一次编译后，将机器码保存下来，下次直接使用（解决解释器逐行解释执行效率低与热点代码经常被调用的问题）
  • 根据代码每次执行情况做一些优化，执行次数越多，速度越快
  • JDK9引入了AOT编译模式，将字节吗直接编译成机器码，避免了JIT预热等方面的开销（但编译质量相对较差），JDK支持分层编译和AOT协同使用

Oracle JDK 与 Open JDK区别

  • Open JDK是一个参考模型，完全开源，而Oracle JDK是Open JDK的一个实现，并不完全开源
  • Oracle JDK相对更稳定，有更多的类和错误修复
  • Oracle JDK提供了更好的性能
  • Oracle不会为即将发布的版本提供长期支持
  • Oracle JDK根据二进制代码许可协议获得许可，Open JDK根据GPL v2获得许可

Java编程基础

基本语法

  public static void main(String[] args) {
    // 定义一个字符串变量

  • 功能性语句和结构性语句组成 结构性语句主要是为了承载功能性语句

书写规范

  • 包名：不使用大写的英文字母
  • 类名和接口名：首字母大写
  • 常量：英文字母大写 单词之间使用下划线分割
  • 变量名和方法名：驼峰命名

注释

  • 单行注释
  • 多行注释
    • 文档注释，一般写在方法或类上

数据类型

Java要确定每种基本类型所占存储空间的大小，并不随着机器硬件架构的变化而变化，因此更具备可移植性。

  • 整型
    • byte 8位
    • short 16位
    • int 32位
    • long 64位
  • 浮点数
    • float 32位 +f
    • double 64位 +d
  • 字符（相当于一个整型ASCII值，可参加表达式运算）
    • char 16位，2个字节
  • 字符串（代表一个地址值）
    • String
  • 布尔常量
    • boolean
      • true
      • false

变量

  • 程序运行期间会产生一些临时数据，保存在一些内存单元中，用一个标识符来表示，称之为变量
  • 声明变量时候要指定变量的数据类型 赋值时候需要相互匹配

类型转换

  • 自动类型转换
    • 两种数据彼此兼容
    • 目标类型的取值范围大于源类型取值范围
    • 例如：byte类型转换至short, int, double等
  • 强制类型转换
    • 两种类型彼此不兼容
    • 目标类型的取值范围小于源类型
    • 会造成精度丢失
    • 格式：目标类型 变量 = （目标类型）值

作用域

  • 变量需要在作用范围内才可以被使用
  • 一般定义在某一对大括号中

引用

Java是按值传递参数，在将一个参数传入一个方法时，本质上是将对象的地址以值的方式传递到形参中

  • 方法得到的是所有参数值的一个拷贝，不能修改传递给它的任何参数变量内容
  • 对象引用作为参数则指向通过同一个对象（如数组），外部引用对象的改变会反映到所对应的对象上
  • 一个方法可以改变一个对象参数的状态
  • 一个方法不能让对象参数引用一个新的对象
